## Meeting Topics

- Root cause and solution to the "mixed node" problem: [Indy Node issue 1769](https://github.com/hyperledger/indy-node/issues/1769)
- Discussed [https://github.com/hyperledger/indy-node/pull/1783](https://github.com/hyperledger/indy-node/pull/1783)
  
  - [Christian Bormann](https://lf-hyperledger.atlassian.net/wiki/people/712020:402bd53a-7b29-43cf-927d-955c323c7ed7?ref=confluence) would appreciate some extra eyes on the code.  [Char Howland](https://lf-hyperledger.atlassian.net/wiki/people/60998bf1dafdf00068e21bae?ref=confluence) indicated she and possibly [Daniel Bluhm](https://lf-hyperledger.atlassian.net/wiki/people/712020:c322d585-d6d2-4479-a990-b91fac45db1c?ref=confluence) would take a look.
- Need a way to test the network migration (16.04 to 20.04) on test networks affected by the ordering issue.
  
  - Testing to include migration and then switching to sorted txn ordering.
  - [Christian Bormann](https://lf-hyperledger.atlassian.net/wiki/people/712020:402bd53a-7b29-43cf-927d-955c323c7ed7?ref=confluence) indicated he would look into creating a tool to write affected transactions to a network.  This would allow testing on pure disposable indy-node networks.
